New complexity introduced a new bias — v12 initially showed 327.53% and 94.24% win rate. Root cause: Kalman β drift between entry and exit was being counted as position P&L. ## What Changed Since Feb 21

The Feb 21 post identified two root causes of OOS degradation: stale hedge ratios and a permissive signal threshold. Three changes targeted both:

| Change | Implementation |
| ------ | -------------- |
| Kalman filter dynamic hedge ratios | 1D Kalman filter estimates β_t at each timestep; replaces fixed β=1 log-price spread in `calculate_spread()` |
| Raised signal thresholds | Z-score entry 1.5→2.0 in signal gate + main loop hardcoded threshold 1.8→2.0; signal strength 0.60→0.65; confidence 0.65→0.70 |
| CUSUM structural break detection | Page-CUSUM on last 63 days of spread before each trade entry; skips pair if break detected |

**Kalman filter detail:** Models log(p1)_t = β_t × log(p2)_t + ε_t with β_t as a random walk (β drifts slowly). Process noise Q = δ/(1−δ) × var(log p2), δ=1e-5 (β can drift ~1% per year). Measurement noise R=0.001. β initialised via OLS on first 30 observations to avoid burn-in. Spread = log(p1)_t − β_t × log(p2)_t.

**CUSUM detail:** Two-sided Page-CUSUM on standardised first differences of the spread window. Threshold=5.0, k=0.5 (standard values, equivalent to detecting a sustained ~5σ shift). Returns True if structural break detected → pair skipped for the day.

## New Bias Found: Kalman β Drift Counted as P&L

**Symptom:** v12 (pre-fix) reported 327.53% return, Sharpe 13.01, win rate 94.24%, max drawdown −0.29%. All 19 walk-forward windows profitable. Quant HF fund comparison: +7,813%. Recognised as the same pattern as v6 (14,739%), v8 (499.6%).

**Root cause:** The spread is defined as:

```
spread_t = log(p1_t) − β_t × log(p2_t)
```

The Kalman filter updates β continuously. Between trade entry (date) and exit (next_date), β drifts from β_entry to β_exit.

The P&L formula was:
```
spread_return = spread_exit − spread_entry
             = [log(p1_exit) − β_exit × log(p2_exit)]
             − [log(p1_entry) − β_entry × log(p2_entry)]
             = Δlog(p1) − β_exit × Δlog(p2) − (β_exit − β_entry) × log(p2_entry)
```

The term `−(β_exit − β_entry) × log(p2_entry)` is a phantom P&L from β drift alone. For stocks priced above $1, log(p2_entry) > 0. When β increases post-entry (as the Kalman filter calibrates to the current regime), this term is negative — the measured spread falls toward zero. The system records a winning trade. Prices did not move.

The exit condition had the same flaw. Exit was triggered when the Kalman z-score dropped below 0.5. The z-score was computed using the *current* β at each candidate exit date. As β drifted, the z-score mechanically fell — triggering exits from β adaptation, not from actual price mean reversion. This is why 94.24% of trades "won": most exits were β-drift exits, not reversion exits.

**Fix:** Lock β at the entry date. Two changes in `trading_system.py`:

1. **Exit condition** — compute locked spread at each exit candidate: `locked_spread_t = log(p1_t) − β_entry × log(p2_t)`. Compute z-score from this locked spread (not the current Kalman spread). β_entry recovered at entry as: `β_entry = (log(p1_entry) − current_spread) / log(p2_entry)`.

2. **P&L calculation** — replace `next_spread − current_spread` with actual price log returns:
```python
log_ret1 = log(p1_exit / p1_entry)
log_ret2 = log(p2_exit / p2_entry)
spread_return = log_ret1 − log_ret2   # action==0: long p1, short p2
spread_return = log_ret2 − log_ret1   # action==2: long p2, short p1
```
This matches the actual equal-dollar position held, regardless of what the Kalman β shows at exit.

The Kalman filter is still used for signal generation (entry z-score). The β-lock only affects exit timing and P&L measurement.

**Note on CUSUM and Kalman:** After the β-lock fix, CUSUM turned out to have minimal effect on Kalman spreads. The Kalman filter is designed to make spread innovations approximately i.i.d. (white noise) — by construction, CUSUM rarely detects a structural break in the first differences of a well-fitted Kalman spread. CUSUM remains in the code as a safety check but is largely inactive in this implementation.

## Honest Interpretation

The 29.34% main backtest is technically unbiased. The walk-forward OOS of +5.41%/quarter is the stricter test — that is 21.6%/year from the OOS windows.

Wait. That sounds high. Context: the walk-forward OOS *stitched* number benefits from compounding individual quarterly returns. The per-window averages are genuinely +5.41%/quarter but include some very strong individual windows (W11 +10.91%, W12 +11.56%, W14 +25.74%) alongside negative ones (W15 −4.35%, W18 −1.50%, W19 −9.10%). The last three OOS windows trend lower — the most recent data is the weakest.

The real production expectation, accounting for realistic mid-cap costs and the late-2024/2025 deterioration visible in W15, W18, W19: probably 5–10% annualised at best, negative at leveraged or high-cost profiles. The signal exists. It is not large enough to survive realistic friction.
